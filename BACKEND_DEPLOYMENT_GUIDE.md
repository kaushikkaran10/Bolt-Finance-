# 🔧 Backend Deployment Guide

## Overview
Your FastAPI backend requires:
- PostgreSQL database
- MongoDB instance
- Redis cache
- Python 3.11+

**These cannot run on Vercel** (Vercel is frontend-only). Deploy backend separately to Railway, Render, or your own server.

---

## ✅ Option 1: Railway.app (Recommended)

### Why Railway?
✅ Simplest setup
✅ Supports Docker & Python directly
✅ Free tier available
✅ Auto-deploys on Git push
✅ Built-in PostgreSQL, MongoDB, Redis support

### Setup Steps

#### 1. Create Railway Account
- Go to https://railway.app
- Sign up with GitHub (recommended)

#### 2. Create New Project
1. Click "Create New Project"
2. Select "Deploy from GitHub"
3. Authorize GitHub and select your repository

#### 3. Add Services

**PostgreSQL Service:**
1. Click "Add Service" → "Add from Marketplace"
2. Search "PostgreSQL"
3. Select and click "Add"
4. Railway creates a managed PostgreSQL instance

**MongoDB Service:**
1. Click "Add Service" → "Add from Marketplace"
2. Search "MongoDB"
3. Select and click "Add"

**Redis Service:**
1. Click "Add Service" → "Add from Marketplace"
2. Search "Redis"
3. Select and click "Add"

#### 4. Deploy Backend Service
1. Click "Add Service" → "GitHub Repo"
2. Select your repository
3. Set root directory to `backend`
4. Click "Deploy"

#### 5. Configure Environment Variables

In Railway dashboard, go to **Variables** tab and add:

```
# PostgreSQL (auto-generated by Railway)
DATABASE_URL=${{Postgres.DATABASE_URL}}

# MongoDB (auto-generated by Railway)
MONGODB_URL=${{MongoDB.DATABASE_URL}}

# Redis (auto-generated by Railway)
REDIS_URL=${{Redis.DATABASE_URL}}

# App Config
APP_NAME=NovaX
DEBUG=false
JWT_SECRET=your-secure-random-string-here-64-chars-minimum
JWT_ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=15
REFRESH_TOKEN_EXPIRE_DAYS=7

# CORS - UPDATE WITH YOUR VERCEL DOMAIN
CORS_ORIGINS=["https://your-app.vercel.app", "http://localhost:3000"]

# External APIs
GOOGLE_API_KEY=your-google-generative-ai-key
NEWS_API_KEY=your-newsapi-key
SENTRY_DSN=your-sentry-dsn-optional
```

#### 6. Buildpack Configuration (if needed)

Create `backend/Procfile`:
```
web: uvicorn main:app --host 0.0.0.0 --port $PORT
```

#### 7. Deploy & Monitor
- Railway auto-deploys on `main` branch push
- Check deployed service for public URL
- View logs in Railway dashboard

## ✅ Option 2: Render.com

### Why Render?
✅ Free tier available
✅ Good for prototyping
✅ Simple deployment
⚠️ Free tier limitations

### Setup Steps

1. **Create Account**: https://render.com/signup

2. **Create Web Service**:
   - Click "New+" → "Web Service"
   - Select "Build and deploy from Git"
   - Connect GitHub
   - Select repository
   - Set root directory: `backend`

3. **Configuration**:
   - **Environment**: Python 3.11
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `uvicorn main:app --host 0.0.0.0 --port $PORT`

4. **Add PostgreSQL Database**:
   - In Render dashboard, create new PostgreSQL
   - Copy internal database URL

5. **Add Environment Variables**:
   ```
   DATABASE_URL=your-postgres-url
   MONGODB_URL=your-mongodb-url
   REDIS_URL=your-redis-url
   JWT_SECRET=secure-random-string
   CORS_ORIGINS=["https://your-app.vercel.app"]
   GOOGLE_API_KEY=your-key
   NEWS_API_KEY=your-key
   ```

6. **Deploy**: Click "Create Web Service"

## ✅ Option 3: Docker on Your Own Server

### Create Dockerfile

In `backend/Dockerfile`:

```dockerfile
FROM python:3.11-slim

WORKDIR /app

# Install system dependencies
RUN apt-get update && apt-get install -y \
    build-essential \
    && rm -rf /var/lib/apt/lists/*

# Install Python dependencies
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# Copy application
COPY . .

# Expose port
EXPOSE 8000

# Run application
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
```

### Create docker-compose.yml

In root `docker-compose.yml`:

```yaml
version: '3.8'

services:
  postgres:
    image: postgres:15
    environment:
      POSTGRES_DB: novax_db
      POSTGRES_USER: novax
      POSTGRES_PASSWORD: your-secure-password
    volumes:
      - postgres_data:/var/lib/postgresql/data
    ports:
      - "5432:5432"

  mongodb:
    image: mongo:7
    environment:
      MONGO_INITDB_ROOT_USERNAME: admin
      MONGO_INITDB_ROOT_PASSWORD: your-secure-password
    volumes:
      - mongodb_data:/data/db
    ports:
      - "27017:27017"

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"

  backend:
    build:
      context: ./backend
      dockerfile: Dockerfile
    ports:
      - "8000:8000"
    environment:
      DATABASE_URL: postgresql+asyncpg://novax:your-secure-password@postgres:5432/novax_db
      MONGODB_URL: mongodb://admin:your-secure-password@mongodb:27017
      REDIS_URL: redis://redis:6379/0
      JWT_SECRET: your-secure-random-string
      CORS_ORIGINS: ["https://your-app.vercel.app"]
    depends_on:
      - postgres
      - mongodb
      - redis

volumes:
  postgres_data:
  mongodb_data:
```

### Deploy

```bash
# Build and run
docker-compose up -d

# View logs
docker-compose logs -f backend

# Stop
docker-compose down
```

## 🔗 Connect Frontend to Backend

After deploying backend, update Vercel environment variables:

1. Go to Vercel Dashboard → Your Project → Settings
2. Update Environment Variables:
   ```
   NEXT_PUBLIC_API_URL=https://your-railway-backend-url.railway.app
   BACKEND_URL=https://your-railway-backend-url.railway.app
   ```
3. Trigger new deployment (or wait for auto-redeploy)

## 🚨 Common Issues

### PostgreSQL Connection Timeout
→ Check connection string
→ Verify firewall rules
→ Ensure PostgreSQL service is running

### MongoDB Connection Error
→ Verify MongoDB URL format
→ Check credentials
→ Ensure MongoDB service is accessible

### CORS Errors from Frontend
→ Add Vercel domain to `CORS_ORIGINS` in backend `.env`
→ Restart backend service

### Dependencies Installation Fails
→ Check Python version (should be 3.11+)
→ Verify all dependencies in `requirements.txt` are compatible
→ Check for system dependencies (e.g., build-essential)

### High Memory Usage
→ Reduce TensorFlow/ML model footprint
→ Deploy ML models separately if needed
→ Use model quantization

## 🔐 Security Checklist

- [ ] `JWT_SECRET` is truly random and strong
- [ ] Database passwords are not in `.env` files (use service-generated URLs)
- [ ] CORS is restricted to your Vercel domain
- [ ] No API keys committed to Git
- [ ] Use `.env` files locally only
- [ ] All sensitive data marked as "Secret" in deployment platform
- [ ] HTTPS enabled on all connections
